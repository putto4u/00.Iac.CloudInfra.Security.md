보안 실습 환경 구축의 첫 단추인 **VirtualBox 6.0.12** 설치 및 최적화 가이드를 정리해 드립니다. 해킹 보안 실습은 일반적인 가상화와 달리 네트워크 격리와 하드웨어 제어가 핵심입니다.

---

## 1. VirtualBox 6.0.12 설치 및 무결성 검증

보안 전문가라면 소프트웨어 설치 전 **공식 출처** 확인과 **무결성** 체크가 기본입니다.

* **공식 다운로드 페이지:** [VirtualBox 6.0 Old Builds](https://www.virtualbox.org/wiki/Download_Old_Builds_6_0)
* **필수 다운로드 항목:**
1. **플랫폼 패키지:** `VirtualBox-6.0.12-133076-Win.exe` (Windows 호스트 기준)
2. **확장팩(Extension Pack):** `All supported platforms` (버전 번호가 6.0.12인지 반드시 확인)


* **실전 팁:** 기업망이나 연구실 환경에서는 설치 파일의 해시값(SHA256)을 비교하여 파일 변조 여부를 확인하는 습관을 가지세요.

---

## 2. 확장팩(Extension Pack) 설치가 필수인 이유

많은 입문자가 확장팩 설치를 간과하지만, **보안 실습**에서는 선택이 아닌 필수입니다.

1. **USB 2.0/3.0 지원:** 무선 해킹 실습을 위해 외부 'USB 무선 랜카드'를 가상 머신(Kali Linux 등)에 직접 연결할 때 필요합니다.
2. **RDP(Remote Desktop Protocol) 지원:** 가상 머신에 원격으로 접속하여 침투 테스트를 수행하는 환경을 구축할 때 사용됩니다.
3. **디스크 암호화:** 가상 머신 내의 민감한 실습 데이터를 보호하기 위해 가상 디스크 자체를 암호화하는 기능을 제공합니다.
4. **NVMe 지원:** 최신 고성능 스토리지 인터페이스를 가상 환경에서 구현할 때 활용됩니다.

## 3. 단계별 실습 환경 구축 (Step-by-Step)

### 단계 1: 호스트 엔진 설치

1. 설치 파일을 실행하고 `Custom Setup` 단계에서 모든 항목(특히 **Networking**)이 선택되었는지 확인합니다.
2. 설치 중 네트워크가 일시 단절된다는 경고창이 뜨면 `Yes`를 클릭합니다. (가상 네트워크 드라이버 설치 과정)

### 단계 2: 확장팩 적용 (중요)

1. VirtualBox 실행 후 `파일(F)` > `환경 설정(P)` > `확장` 탭 클릭.
2. 오른쪽 우측의 `01` 모양 아이콘(패키지 추가)을 눌러 다운로드한 확장팩 파일을 선택합니다.
3. 스크롤을 끝까지 내려야 `동의` 버튼이 활성화됩니다. **(전문가 팁: 라이선스 내용을 읽는 척이라도 하며 끝까지 내리는 것이 포인트입니다.)**

### 단계 3: 가상 네트워크 브릿지 설정

* **NAT:** 외부 인터넷 연결이 필요할 때.
* **Host-Only:** 외부와 격리된 '샌드박스' 환경에서 악성코드를 분석하거나 내부망 침투 실습을 할 때 필수 설정입니다.

## 4. ⚠️ 자주 오해하거나 실수하는 부분 (주의!)

> **❌ 실수 1: BIOS에서 가상화 기술(VT-x/AMD-V) 미활성화**
> * **현상:** 가상 머신 실행 시 '64-bit' 옵션이 안 보이거나 에러가 발생합니다.
> * **해결:** PC 재부팅 시 F2/Del 키를 눌러 BIOS 진입 후, `Virtualization Technology`를 **Enabled**로 변경해야 합니다.
> 
> 

> **❌ 실수 2: 게스트 확장(Guest Additions)과 확장팩(Extension Pack) 혼동**
> * **구분:** **확장팩**은 호스트 프로그램에 설치하는 것이고, **게스트 확장**은 가상 머신(OS)을 설치한 '내부'에서 화면 해상도나 클립보드 공유를 위해 설치하는 도구입니다. 둘 다 설치해야 원활한 실습이 가능합니다.
> 
>

## 5. IT 보안 전문가의 실전 팁 (Pro-Tips)

1. **스냅샷(Snapshot)의 생활화:** 취약점 공격(Exploit) 성공 후 시스템이 뻗는 경우가 많습니다. **OS 설치 직후**, **도구 설정 직후**에 반드시 스냅샷을 찍으세요. "과거로 돌아가는 마법"입니다.
2. **네트워크 카드 분리:** 공격자 VM과 희생자 VM을 동일한 `Internal Network`로 묶어 실제 인터넷 망에 공격 트래픽이 나가지 않도록 설계하는 것이 프로의 자세입니다.
3. **VT-x 에러 대응:** Windows 10/11의 'Core Isolation(핵심 격리)' 기능이 켜져 있으면 VirtualBox와 충돌할 수 있습니다. 에러 발생 시 이 기능을 잠시 끄는 것을 검토하십시오.
